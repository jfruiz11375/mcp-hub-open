# Contributing

Thanks for contributing to MCP Hub Open.

## Development principles

- Keep the MVP simple and easy to run locally
- Prefer readable code over clever abstractions
- Make security tradeoffs explicit in PRs
- Add docs when behavior changes

## Local development

```bash
npm install
npm run dev
```

## Pull request checklist

- [ ] The change solves a real user problem
- [ ] The change is documented in README or inline docs
- [ ] New config is added to `.env.example` when needed
- [ ] Commands and process execution changes are explained clearly

## Good first issues

- Better repo validation
- Add Docker runner
- Add test coverage
- Add tool discovery page
- Add encrypted secret storage
